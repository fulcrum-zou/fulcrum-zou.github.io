Git同步fork的仓库

2023-02-02

上周上MHRI的时候把fork过来的仓库clone到了本地，今天需要把原仓库更新的内容同步过来，直接在网站上操作的话只能放弃之前的commits，所以这里记录一下怎么在本地操作.

0 查看远程仓库信息

1 将原仓库添加到remote

取名为upstream.

查看是否添加成功.

2 同步fork

从原仓库fetchbranch和commits并传送到本地，此时被存储在了本地分支upstream/main中.

确保切换到了本地分支.

3 合并分支

把upstream/main合并在本地的master上

发现有冲突.

如果没有冲突可以跳过第4步.

4 合并冲突

5 推送到远程仓库

6 下次更新

直接从第2步开始.
